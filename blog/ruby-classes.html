Ruby Classes

1.15.2016

When working with objects that share similar characteristics but are different from each other, classes can offer simplicity and organization. Classes can hold a number of variables and methods that are specific to that class object. Let's make a class to keep track of all of the books in your library! We start by defining our new class and what it should do when we make an instance of one. We can also define the title and author when we create a new book. Classes use both local and instance variables. Local variables, as their name suggests, are limited to the method they're a part of. Instance variables however are available to the entire instance or specific object of that class. Instance variables are denoted with the @ symbol. 

class Books 
 def initialize(title, author) 
  @title = title 
  @author = author 
 end 
end
Here we have created our class Books, which requires two arguments upon creation. It then sets instance variables those arguments equal to the supplied arguments. To call an instance of our new class. type: 
first_book = Books.new("Ulysses", "James Joyce")

Our class doesn't do much right now, so let's add some more to it! Instance variables are shared across the object, so let's create some methods to access their value so other methods can reference them as well. We'll do this with a rather simple method that looks like this: 

def title 
 @title
end

def author 
 @author 
end
Now whenever you call the method title, it will return @title and similarly for author.

Okay, that's great but our class still doesn't do much. 

def info 
 puts title + "," + author 
end
Now we can call our info on a class we've created and it will print out the title and author 
first_book.info
Like instance variables, these are instance methods. They are available to the entire instance of this particular class. As you can imagine we could create more methods to add more information to our class like publisher, copyright date, plot, etc.
