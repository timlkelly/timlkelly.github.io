Regular Expressions

1.30.2016

Regular expressions, also know as regexp or regex, is a sequence of characters that is used to define a search pattern. They are mainly used for searching through strings by pattern matching. In Ruby the regex constructor is //. This is an empty expression but the pattern would go in between the forward slashes. The method match can be used to detect matches in a string. Match is both a regex and string method so it can be used either way. 

puts "Match!" if /cat/.match("There's a cat in this string!")
puts "Match!" if "There's a cat in this string!".match(/cat/)
Ruby also provides a regex operator: =~. 
puts "Match!"if "There's a cat in this string!" =~ /cat/
Though the =~ operator will return the numerical index of the character in the string where the match started. Match will return an instance of the class MatchData:

Here are some additional regular expression search parameters.

. is the wild card character c.t would match "cat"

Square brackets [] can be used can be used to match a single character or through a range. [cat] would match the characters "c", "a", and "t". [a-g] would match any lower case character a through g. [cb]at would match "cat" and "bat".

[^ ] matches a single character that is not inside of the brackets. [^cat] will match any character other than "c", "a", or "t".

^ before the pattern will match if the starting position or line of the string begins with that pattern. $is the opposite of ^. It looks for matches at the end of the string

/^cat/.match("Hello cat") would not match cat but /$cat/.match("Hello cat")would.

() allows for submatches or subexpressions. These can later be recalled with $n where n refers to the subexpression that was set. If you were to use a regular expression with multiple subexpressions you could recall the first one with $1, $2 would recall the second subexpression and so on.

{} specifies exactly how many repetitions of the pattern to match by providing a number inside the curly braces directly after the expression to be matched. /\d{3}/ will match exactly three digits. Ranges can be specified like so {3,7} which would return any matches between three to seven and lastly {3,} results in three or more.

* and + are the zero or more and one or more quantifiers, respectively. They will return as much of the string as they possibly can. Adding a ? after either quantifier will make them stop at the end of the first match.

Regular expressions seem like powerful tools but they can easily become overwhelming to decipher. What do you imagine \b(\w)?(\w)\w?\2\1 does? Or how about \b\d{3}[-.]?\d{3}[-.]?\d{4}\b? Well the first one will return two to five letter palindromes and the second returns phone numbers that use hyphons, periods or do not contain a spacer. As with anything, I'm sure you can read these faster as you use regex more and more, however it does provide for a rather steep learning curve.

regexr.com is a great interactive website to learn and experiment with regular expressions. Check it out!
